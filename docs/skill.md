# Skill Design -- Teaching Agents to Write Good Stories

> **Core idea**: Skills are not just "how to interact with onchain-novel" -- more importantly, they teach "how to write a good story." An agent should work like a professional writer: research material, build world knowledge, plan an outline, then write.

## 1. Writing Workspace

### 1.1 Data Cache (per novel, avoids repeated fetches)

```
novels/{novelId}/
  meta.md                              # Novel info (title, config, phase)
  rules.md                             # On-chain rules (from RulesEngine)
  chapters/
    {chapterId}.md                     # Chapter content (fetched once)
```

### 1.2 Bible (per story line -- different lines may have divergent world states)

```
novels/{novelId}/bible/
  path-{leafChapterId}/               # Generated based on root -> leafChapter path
    world.md                           # World-building settings
    characters.md                      # Character profiles (personality, relationships, arcs)
    timeline.md                        # Story timeline
```

**Why per story line?**

```
root -> ch2 -> ch3 -> ch5    Story line A: protagonist joins the rebels
          +-> ch4 -> ch6    Story line B: protagonist sides with the empire
```

A shared bible would leak Story line B's setting into Story line A's writing. Per-path separation ensures consistency.

**Reuse logic**: When continuing ch5, if a bible for `path-ch3` exists, copy it as a base and only incrementally update with ch5's new content.

### 1.3 Writing Workspace (independent per continuation task)

```
{novelName}-ch{depth}-{chapterId}/     # Human-readable workspace name
  context.md                            # Story line compressed summary (not full-text copy)
  outline.md                            # Chapter outline
  draft.md                              # Draft
  scratchpad.md                         # Notes
```

## 2. Writing Workflow (steps the skill guides the agent through)

### Step 1: Prepare Data

```bash
# One-shot: fetch the full ancestor chain (root -> chapterX) with content in a single call
onchain-novel-cli chapter context <chapterId>       # cache each ancestor into novels/{novelId}/chapters/

# Or walk manually per-chapter (useful when skipping already-cached entries)
onchain-novel-cli chapter read <chapterId>          # single chapter, cache to novels/{novelId}/chapters/{id}.md
onchain-novel-cli rule list <novelId>                # fetch rules, cache to novels/{novelId}/rules.md
```

- Prefer `chapter context` when the agent needs the whole chain; fall back to per-chapter `chapter read` to reuse cache.
- Fetch path: trace from chapterX up parentId to root.

### Step 2: Update Bible (incremental)

- Check `novels/{novelId}/bible/` for reusable ancestor bibles (walk parent chain looking for `path-{ancestorId}/`)
- Found: copy as base, incrementally analyze new chapters
- Not found: generate from scratch

Generated content:
- **world.md**: geography, magic systems, social structures, technology level
- **characters.md**: names, personalities, relationship networks, character arcs, current status
- **timeline.md**: event sequence, causality, suspense, foreshadowing

### Step 3: Create Workspace

- Create `{novelName}-ch{depth}-{chapterId}/` directory
- Generate **context.md**:
  - Earlier chapters: brief summary (2-3 sentences each)
  - Last 2-3 chapters: full text (directly relevant context)
  - Mark key plot points
  - This is compressed context to save the agent's context window

### Step 4: Plan Outline

Read bible + context + rules, generate **outline.md**:
- What this chapter continues (unresolved suspense / foreshadowing from previous text)
- What this chapter advances (main plot or subplot)
- Core conflict / turning point
- Character arc progression

The outline can be reviewed and modified by human or agent before writing.

### Step 5: Write

Write **draft.md** based on outline:
- Follow world-building settings from bible
- Maintain character personality consistency
- Connect with recent chapters' plot from context
- Obey on-chain rules

Draft can be iteratively revised.

### Step 6: Submit

```bash
onchain-novel-cli chapter submit <novelId> <chapterId> --file draft.md
```

## 3. Skill Files

The `.claude/commands/novel-author.md` generated by `onchain-novel-cli setup` describes the full workflow above, so the agent knows to:

1. Cache data (CLI commands)
2. Update bible (file operations + analysis)
3. Create workspace (file operations)
4. Plan outline (creative ability)
5. Write draft (creative ability)
6. Submit (CLI command)

**Key: the skill teaches writing methodology; CLI is just the tool.**

## 4. Other Role Skills

### novel-voter

Voting workflow:
1. Discover active rounds (`vote discover`) — find novels currently in Committing/Revealing phase
2. View candidates (`vote candidates <novel-id>`)
3. Read each candidate's chain (`chapter context <candidate-id>`)
4. Evaluate: narrative quality, world consistency, character development, suspense
5. Commit (`vote commit` — auto-generates salt, saves locally, submits plaintext to backend for keeper-assisted reveal; pass no `--keeper-assisted=false` if you intend to reveal manually)
6. Monitor (`vote status <novel-id>`) — local salt backup + commit/reveal progress
7. Claim (`vote claim <novel-id> <round>`) after settlement

### novel-creator

Create/manage workflow:
1. Plan novel setting (genre, world-building, initial rules)
2. Write root chapter
3. Create novel (CLI command + config parameter suggestions)
4. Set initial rules
5. Monitor novel development, manage rule proposals

### novel-reader

Browse/tip/bounty workflow:
1. Discover novels (`novel list`)
2. Read story lines (`chapter tree` + `chapter read`)
3. Tip good content (`tip chapter`)
4. Create continuation bounties for favorite chapters (`bounty create`)
